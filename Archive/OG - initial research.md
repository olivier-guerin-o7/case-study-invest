### Q1: What is a Brokerage Account? Do we deep dive into its settings?

A **brokerage account** (French: *Compte-Titres Ordinaire / CTO*) is a standard investment account that lets you buy and sell financial instruments — stocks, ETFs, bonds, funds. Unlike a PEA (which has tax advantages but restrictions), a CTO has no cap, no holding period, and no geographic restrictions. You can buy Apple, a Nasdaq ETF, or a Japanese bond.

In Finary's context, this means: users will be able to **place buy/sell orders for stocks and ETFs directly inside the Finary app**, instead of going to a separate broker (Degiro, Trade Republic, etc.).

**Do we deep dive into settings?** No — but we go further than "just leading to it." Here's my read of the assignment scope:

The assignment says: *"Design the experience for a user's first investment; from the moment they open the app to **completing their first trade**."*

"Completing" = the trade happens. So the scope is:

**Dashboard** → **Discovery/Decision** (what to buy) → **Order Screen** (how much, at what price) → **Confirmation** (review + legal) → **Execution feedback** (done!) → **Post-trade reinforcement**

We do NOT need to design: account settings, deposit/withdrawal flows, portfolio rebalancing, recurring investments, sell flows. Those are "after first trade" concerns.

**What we DO need to show on the order screen:**
- Asset selection (which ETF/stock)
- Amount (€ or shares)
- Order type (market order is fine for v1 — simplest for a hesitant user)
- Fee transparency (MiFID II requires this)
- Confirm button with clear terms

This is 1-2 screens, not a deep dive. But it must feel trustworthy and clear.

### R3-17: Blueprint Wireframes — Can I Do It?

**Yes.** Here's how:

I can create a CSS "blueprint mode" that renders our hi-fi components as line-drawn wireframes. Think:
- White/light blue outlines on dark blue background (classic blueprint aesthetic)
- No fills, no photos, no colors — just structure
- Text becomes placeholder labels ("Header", "CTA", "Sparkline chart")
- Icons become simple outlined shapes
- Subtle grid lines in the background

**Technically**, this is a CSS theme applied to the same React components. We build the hi-fi version first, then add a `.blueprint` class that strips everything down to outlines. Same layout, same proportions, completely different feel.

**Two approaches:**

1. **CSS filter trick** (fast): Apply a combination of `grayscale(1)`, custom `outline` styles, and `mix-blend-mode` to approximate the blueprint look. Quick but less refined.

2. **Dedicated blueprint theme** (better): A separate Tailwind color palette where all backgrounds → transparent, all fills → stroke-only, all text → monospace, all images → placeholder boxes with crosshatch. More work, but genuinely beautiful.

I'd go with **approach 2** for the user flow storyboard — it's a small number of mini-screens (5-7 thumbnails) so the effort is contained, and the visual impact is high. It's the kind of detail that makes someone stop and think "this person cares about presentation."
